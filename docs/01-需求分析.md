# 五子棋机器人 — 需求分析

> 版本: V0.1 | 日期: 2026-03-19

## 1. 项目背景

利用 Hugging Face LeRobot 开源框架和 SO-100 桌面机械臂，搭建一个能与人类面对面下五子棋的机器人系统。项目目标不仅是实现对弈功能，更是作为 LeRobot 框架在实际操控任务中的技术验证和团队能力建设。

**为什么选五子棋：**
- 规则简单（五连胜），无吃子/移子，只有"放"的动作
- 棋子形状统一（圆形扁平），黑白两色，视觉识别难度低
- 棋盘为规则网格，位置离散且有限
- 但仍然涉及完整的 感知→决策→执行 闭环

## 2. 功能需求

### 2.1 核心功能

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F1 | 棋盘状态感知 | 通过摄像头识别棋盘上每个交叉点的状态（空/黑/白） | P0 |
| F2 | AI 落子决策 | 根据当前棋盘状态计算最优落子位置 | P0 |
| F3 | 棋子抓取 | 从棋子托盘中拾取一颗己方棋子 | P0 |
| F4 | 棋子放置 | 将棋子准确放置到目标交叉点上 | P0 |
| F5 | 对弈流程控制 | 交替走棋、胜负判定、游戏结束处理 | P0 |
| F6 | 落子验证 | 放置后重新拍照验证棋子是否到位 | P1 |
| F7 | 对手走棋检测 | 自动检测人类落子（无需手动按键） | P1 |
| F8 | 失败恢复 | 抓取失败或落子偏差时的重试机制 | P1 |
| F9 | 难度调节 | AI 搜索深度可调，适应不同水平对手 | P2 |
| F10 | 可视化界面 | 实时显示棋盘状态、AI 思考过程、相机画面 | P2 |

### 2.2 非功能需求

| ID | 需求 | 指标 |
|----|------|------|
| NF1 | 落子精度 | 棋子中心距目标交叉点 < 5mm |
| NF2 | 视觉识别准确率 | > 98%（标准光照条件下） |
| NF3 | 单步操作时间 | 拾取+放置 < 15s |
| NF4 | AI 思考时间 | < 3s（9×9 棋盘） |
| NF5 | 抓放成功率 | > 90%（初期），> 95%（调优后） |
| NF6 | 连续运行稳定性 | 完成一盘棋（~40 步）无崩溃 |

## 3. 用户场景

### 场景 1: 标准对弈（MVP）

```
1. 开机，机械臂自动回到安全位 (home)
2. 系统拍照确认棋盘为空，提示"游戏开始"
3. AI（黑方）先手:
   a. 拍照 → 识别棋盘
   b. AI 计算落子
   c. 机械臂从托盘拾取黑子
   d. 放置到目标位置
   e. 机械臂回到安全位
   f. 拍照验证落子
4. 提示"轮到你了"
5. 人类放置白子
6. 按 Enter（或系统自动检测到新棋子）
7. 重复 3-6 直到分出胜负或平局
```

### 场景 2: 标定流程

```
1. 运行标定脚本
2. 用 leader 臂触碰棋盘四个角的交叉点
3. 系统自动计算坐标变换
4. 验证: 系统在图像上叠加网格，人工确认对齐
5. 保存标定参数
```

### 场景 3: 示教数据采集

```
1. 设定本次目标棋盘位置
2. 在托盘放入棋子
3. 用 leader 臂示教: 拾取 → 移动 → 放置 → 松开 → 缩回
4. 系统自动录制关节轨迹 + 相机画面 + 目标位置
5. 重复，覆盖不同棋盘区域
6. 训练 ACT 策略
```

## 4. 系统边界

### 在范围内

- 单臂机械臂对弈
- 9×9 棋盘（标准五子棋为 15×15，先从小棋盘开始）
- 固定光照、固定棋盘位置
- 人类对手手动放子

### 不在范围内（后续可扩展）

- 双臂操作
- 15×15 标准棋盘
- 棋子自动补给（自动从棋盒取子）
- 语音交互 / 表情反馈
- 在线对弈（连接网络棋类平台）
- 其他棋类（围棋、中国象棋、国际象棋）

## 5. 关键约束

| 约束 | 说明 |
|------|------|
| 机械臂工作空间 | SO-100 臂展约 35-40cm，工作空间约 50×50×50cm。棋盘+托盘必须在此范围内 |
| 夹爪能力 | 平行夹爪，扁平棋子（厚度 5-8mm）需要合适的接近角度 |
| 相机视野 | 顶部俯拍必须覆盖完整棋盘，不能被机械臂遮挡 |
| 棋子规格 | 太小夹不住，太大超出网格间距。推荐直径 15-20mm |
| 环境光照 | HSV 阈值检测对光照敏感，需要稳定的照明条件 |
| LeRobot 版本 | 基于 v0.5.x，API 可能随版本变化 |

## 6. 风险识别

| 风险 | 等级 | 影响 | 应对思路 |
|------|------|------|----------|
| 扁平棋子夹取困难 | 高 | 抓取成功率低 | 棋子加厚/加唇边；夹爪贴橡胶垫；调整接近角度 |
| 落子精度不足 | 高 | 棋子不在交叉点上 | 增加示教数据；工作空间边缘重点采集 |
| 相机标定漂移 | 中 | 视觉检测偏差累积 | 每盘棋开局时自动校验ArUco；检测到偏差则重新标定 |
| 光照变化 | 中 | HSV 阈值失效 | 使用漫射灯罩；动态阈值调整 |
| 棋子堆叠/倾斜 | 中 | 托盘拾取失败 | 使用有序排列的托盘（每槽一子） |
| ACT 策略泛化不足 | 中 | 未见过的位置失败 | 增加数据量；先缩小到 5×5 验证 |
| ROS / LeRobot 环境冲突 | 低 | 依赖安装困难 | 独立 venv，隔离 ROS 的 Python 环境 |

## 7. 验收标准

### MVP（最小可行产品）

- [ ] 棋盘检测准确率 > 95%（标准条件下）
- [ ] 机械臂能在 9×9 棋盘上准确放置棋子（误差 < 5mm）
- [ ] 能完成一盘完整的人机对弈（约 40 步）
- [ ] AI 不会走出明显的低级错误

### 完整版

- [ ] 棋盘检测准确率 > 98%
- [ ] 抓放成功率 > 95%
- [ ] 自动检测对手落子
- [ ] 支持 15×15 标准棋盘
- [ ] 落子失败自动重试
