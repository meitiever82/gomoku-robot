# 五子棋机器人 — 仿真环境搭建指南

> 版本: V0.3 | 日期: 2026-03-25
> 目标机器: 带 NVIDIA GPU (≥8GB VRAM) 的工作站或云实例

## 1. 仿真优先策略

整体路线改为 **Sim-first**：在 Isaac Sim 中完成全流程验证，再迁移到真实 SO-100 硬件。

```
仿真阶段                              硬件阶段
┌─────────────────────────┐          ┌──────────────────┐
│ Isaac Sim + LeIsaac     │          │ SO-100 实物       │
│                         │  迁移    │                   │
│ 场景搭建 → 数据采集        │ ──────▶ │ 标定 → 微调        │
│ → 策略训练 → 联调验证      │         │ → 实物验证         │
└─────────────────────────┘          └──────────────────┘
```

**仿真的优势：**
- 数据采集可自动化（脚本生成轨迹），不需人工示教 50 次
- 无硬件风险，可大胆尝试参数
- GPU 加速并行采集/训练
- Domain randomization 提高 sim-to-real 迁移效果

## 2. 硬件要求

| 项目 | 最低要求 | 推荐 |
|------|---------|------|
| GPU | RTX 4060 Ti (8GB) | RTX 4090 (24GB) / A100 |
| RAM | 32GB | 64GB |
| 磁盘 | 50GB 可用空间 | 100GB+ |
| NVIDIA Driver | ≥ 535.x (CUDA ≥ 12.1) | ≥ 570.x (CUDA ≥ 12.8) |

> **8GB GPU 说明：** RTX 4060 Ti (8GB) 实测可运行，脚本自动启用低显存模式
> (headless + 关闭 DLSS/RTX 后处理 + 摄像头降至 160×120)。物理仿真正常，
> 但摄像头图像质量较低。策略训练和完整渲染建议使用 ≥16GB VRAM。

| 组件 | 版本 | 用途 |

## 5. 一键搭建

项目提供了自动化安装脚本 `scripts/setup_sim.sh`：

```bash
cd gomoku-robot
bash scripts/setup_sim.sh
```

脚本会自动完成：
1. 安装 uv（如未安装）
2. 创建 `.venv-sim` (Python 3.11)
3. 安装 PyTorch + CUDA 12.8
4. 安装 Isaac Sim 5.1.0
5. 安装 LeIsaac + IsaacLab（从本地 `../leisaac/`）
6. 安装 LeRobot 0.4.1
7. 固定 numpy 到 1.26.0
8. 安装本项目
9. 接受 EULA
10. 运行验证

**国内用户** 需要设置 HF 镜像：
```bash
export HF_ENDPOINT=https://hf-mirror.com
bash scripts/setup_sim.sh
```

## 6. 手动搭建（如脚本失败）

### 6.1 安装 uv

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc  # 或 source ~/.zshrc
```

### 6.2 创建虚拟环境并安装依赖

```bash
cd gomoku-robot

# 创建 Python 3.11 虚拟环境
uv venv --python 3.11 .venv-sim
source .venv-sim/bin/activate

# PyTorch + CUDA 12.8
uv pip install torch==2.7.0 torchvision==0.22.0 --index-url https://download.pytorch.org/whl/cu128

# Isaac Sim
uv pip install isaacsim==5.1.0 --extra-index-url https://pypi.nvidia.com

# flatdict 需要特殊处理 (setuptools 在隔离构建中缺失)
uv pip install setuptools
uv pip install --no-build-isolation flatdict==4.0.1

# LeIsaac + IsaacLab (从本地仓库安装)
uv pip install -e '../leisaac/source/leisaac[isaaclab]' --extra-index-url https://pypi.nvidia.com

# LeRobot (LeIsaac 兼容版本)
uv pip install lerobot==0.4.1

# 固定 numpy (被 lerobot 拉高后需要降回来)
uv pip install numpy==1.26.0

# 安装本项目
uv pip install -e .

# 再次固定 numpy (gomoku-robot 依赖可能再次拉高)
uv pip install numpy==1.26.0

# 额外工具
uv pip install opencv-contrib-python pytest
```

### 6.3 验证安装

```bash
source .venv-sim/bin/activate

# 接受 EULA (首次需要)
echo "Yes" | python -c "import isaacsim; print('Isaac Sim OK')"

python -c "import torch; print(f'PyTorch {torch.__version__}, CUDA: {torch.cuda.is_available()}, GPU: {torch.cuda.get_device_name(0)}')"

# 验证 LeRobot

# 验证 gomoku-robot

# 运行 LeIsaac 示例 (SO-101 拾取橙子)
# 8GB GPU 自动启用低显存模式
python scripts/test_sim.py
```

## 8. 仿真阶段工作内容

### 8.1 现有 LeIsaac 环境验证

先用现成的 SO-101 PickOrange 环境跑通整个流程：

```bash
source .venv-sim/bin/activate
python scripts/test_sim.py
```

`test_sim.py` 会根据 GPU VRAM 自动选择模式：
- **≥16GB**: 完整模式（摄像头 640×480，RTX 渲染）
- **<16GB**: 低显存模式（headless，关闭 DLSS/RTX 后处理，摄像头 160×120）

### 8.2 五子棋场景搭建

需要在 IsaacLab 中创建自定义任务环境：

**场景资产 (USD)：**
- 桌面 (已有通用资产)
- 9×9 棋盘 (需要创建: 平面 + 网格纹理)
- 黑/白棋子 (需要创建: 圆柱体, Φ20mm × 8mm)
- 棋子托盘 (需要创建: 带槽位的盒子)
- SO-100 机械臂 (基于现有 SO-101 资产改)

**URDF → USD 转换：**
```bash
# SO-100 URDF 已有: tools/rerun/examples/rust/animated_urdf/data/so100.urdf
# 转换为 USD (IsaacLab 工具)
python -m isaaclab.tools.convert_urdf \
    --urdf_path so100.urdf \
    --usd_path so100.usd
```

**自定义任务环境文件结构：**
```
gomoku_robot/
  sim/
    __init__.py
    gomoku_env_cfg.py       # IsaacLab 任务配置 (场景、奖励、终止条件)
    gomoku_env.py           # Gymnasium 接口封装
    assets/
      gomoku_board.usd      # 棋盘 USD
      gomoku_piece.usd      # 棋子 USD
      piece_tray.usd        # 托盘 USD
```

### 8.3 仿真数据采集

仿真中数据采集可以 **脚本化**，无需人工示教：

```python
# 伪代码: 脚本生成 pick-and-place 轨迹
for episode in range(1000):  # 仿真中可以大量采集
    target_row, target_col = random_board_position()

    # 用 IK 生成参考轨迹
    pick_pose = tray_slot_pose(episode % num_slots)
    place_pose = board_intersection_pose(target_row, target_col)

    trajectory = generate_pick_place_trajectory(pick_pose, place_pose)

    # 在仿真中执行并录制
    record_episode(env, trajectory, target=(target_row, target_col))
```

**优势：** 仿真中可以轻松采集 1000+ 条轨迹（vs 真实中 30-50 条），策略泛化性更好。

### 8.4 策略训练与仿真评估

```bash
# 训练 (和真实数据完全相同的流程)
lerobot-train \
    --policy.type=act \
    --dataset.repo_id=gomoku-sim-pickplace \
    --training.steps=100000 \
    --device=cuda

# 在仿真中评估
lerobot-eval \
    --policy.path=outputs/gomoku-act \
    --env.type=custom \
    --env.hub_path=./gomoku_robot/sim/gomoku_env.py
```

### 8.5 仿真全流程联调

在仿真中跑完整的对弈循环：
1. 仿真相机拍照 → 视觉检测棋盘
2. 引擎算棋
3. ACT 策略控制仿真臂抓放
4. 验证落子
5. 等待"对手"（脚本模拟或手动输入）

## 9. Sim-to-Real 迁移注意事项

仿真验证通过后，迁移到真实硬件时需要注意：

| 差异 | 仿真 | 真实 | 应对 |
|------|------|------|------|
| 物理特性 | 理想刚体 | 摩擦/柔性/误差 | Domain randomization |
| 相机 | 渲染图像 | 真实图像 | 纹理随机化 + 微调 |
| 光照 | 可控 | 变化 | 随机光照条件训练 |
| 夹爪接触 | 简化模型 | 真实摩擦力 | 真实硬件上补充少量示教 |
| 延迟 | 无通信延迟 | USB 串口延迟 | 控制频率匹配 |

**推荐迁移策略：**
1. 仿真中训练基础策略
2. 真实硬件上采集 10-20 条微调数据
3. 在仿真预训练基础上 fine-tune
4. 或直接测试，好的仿真策略有时候 zero-shot 就能迁移

## 10. 已知问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `flatdict` 构建失败 | `pkg_resources` 在 PEP 517 隔离环境中缺失 | 先装 setuptools，再 `uv pip install --no-build-isolation flatdict==4.0.1` |
| Isaac Sim EULA 阻塞 | 首次使用需要交互式接受 | `echo "Yes" \| python -c "import isaacsim"` |
| HF 资产下载超时 | 国内访问 `cdn-lfs-us-1.hf.co` 不稳定 | `export HF_ENDPOINT=https://hf-mirror.com` |
| numpy 版本被拉高 | lerobot/gomoku-robot 依赖拉高 numpy | 安装完所有包后执行 `uv pip install numpy==1.26.0` |
| 8GB GPU OOM (RTX 渲染) | DLSS/RTX 后处理占用大量 VRAM | `test_sim.py` 自动检测并启用低显存模式 |
| CPU powersave 警告 | CPU 限频 | `sudo cpupower frequency-set -g performance` |
| GLFW 初始化失败 | 无显示器 (headless) | 正常现象，不影响仿真 |
| 物理碰撞 convexHull 警告 | 厨房场景三角网格回退 | 正常现象，不影响仿真精度 |
