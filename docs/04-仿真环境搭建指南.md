# 五子棋机器人 — 仿真环境搭建指南

> 版本: V0.1 | 日期: 2026-03-19
> 目标机器: 带 NVIDIA GPU 的工作站

## 1. 仿真优先策略

整体路线改为 **Sim-first**：在 Isaac Sim 中完成全流程验证，再迁移到真实 SO-100 硬件。

```
仿真阶段                              硬件阶段
┌─────────────────────────┐          ┌──────────────────┐
│ Isaac Sim + LeIsaac     │          │ SO-100 实物       │
│                         │  迁移    │                  │
│ 场景搭建 → 数据采集     │ ──────▶ │ 标定 → 微调       │
│ → 策略训练 → 联调验证   │          │ → 实物验证        │
└─────────────────────────┘          └──────────────────┘
```

**仿真的优势：**
- 数据采集可自动化（脚本生成轨迹），不需人工示教 50 次
- 无硬件风险，可大胆尝试参数
- GPU 加速并行采集/训练
- Domain randomization 提高 sim-to-real 迁移效果

### 3.1 安装 Isaac Sim

如果尚未安装，参考 NVIDIA 官方文档：

# 5. LeRobot (LeIsaac 兼容版本)
pip install lerobot==0.4.1

# 6. numpy 版本修复
pip install numpy==1.26.0

# 7. 本项目额外依赖
pip install opencv-contrib-python pytest

python -c "

envs_dict = make_env(
    'LightwheelAI/leisaac_env:envs/so101_pick_orange.py',
    n_envs=1,
    trust_remote_code=True
)

obs, info = env.reset()

### 5.3 仿真数据采集

仿真中数据采集可以 **脚本化**，无需人工示教：

```python
# 伪代码: 脚本生成 pick-and-place 轨迹
for episode in range(1000):  # 仿真中可以大量采集
    target_row, target_col = random_board_position()

    # 用 IK 生成参考轨迹
    pick_pose = tray_slot_pose(episode % num_slots)
    place_pose = board_intersection_pose(target_row, target_col)

    trajectory = generate_pick_place_trajectory(pick_pose, place_pose)

    # 在仿真中执行并录制
    record_episode(env, trajectory, target=(target_row, target_col))
```

**优势：** 仿真中可以轻松采集 1000+ 条轨迹（vs 真实中 30-50 条），策略泛化性更好。

### 5.5 仿真全流程联调

在仿真中跑完整的对弈循环：
1. 仿真相机拍照 → 视觉检测棋盘
2. 引擎算棋
3. ACT 策略控制仿真臂抓放
4. 验证落子
5. 等待"对手"（脚本模拟或手动输入）

## 6. Sim-to-Real 迁移注意事项

仿真验证通过后，迁移到真实硬件时需要注意：

| 差异 | 仿真 | 真实 | 应对 |
|------|------|------|------|
| 物理特性 | 理想刚体 | 摩擦/柔性/误差 | Domain randomization |
| 相机 | 渲染图像 | 真实图像 | 纹理随机化 + 微调 |
| 光照 | 可控 | 变化 | 随机光照条件训练 |
| 夹爪接触 | 简化模型 | 真实摩擦力 | 真实硬件上补充少量示教 |
| 延迟 | 无通信延迟 | USB 串口延迟 | 控制频率匹配 |

**推荐迁移策略：**
1. 仿真中训练基础策略
2. 真实硬件上采集 10-20 条微调数据
3. 在仿真预训练基础上 fine-tune
4. 或直接测试，好的仿真策略有时候 zero-shot 就能迁移
