# 五子棋机器人 — 实施计划

> 版本: V0.4 | 日期: 2026-03-27

## 1. 总体路线（仿真优先 + VLA）

```
Phase 0        Phase 1        Phase 2          Phase 3          Phase 4          Phase 5
准备阶段  ──▶  软件验证  ──▶  仿真环境搭建  ──▶  仿真训练验证  ──▶  硬件迁移  ──▶  实物联调

文档/物料/     引擎/视觉      Isaac Sim        数据采集          标定/微调       完整对弈
环境搭建      单元测试       场景+物理校准     Pi0训练/评估      sim-to-real     调优迭代
(本机)        (本机)        (RTX 4060 Ti)    (Spark 128GB)    (本机+硬件)     (本机+硬件)
```

**与 V0.3 的核心变化：**
- 策略从 ACT 升级为 Pi0 VLA（语言条件化）
- 训练设备从 GPU 机器改为 NVIDIA Spark
- 数据格式新增语言指令字段

### Phase 3: 数据采集与 Pi0 训练 (RTX 4060 Ti + Spark)

**目标：** 生成训练数据，训练 Pi0 VLA 策略，仿真中验证。

| 步骤 | 任务 | 设备 | 预计工作量 |
|------|------|------|-----------|
| 3.1 | 编写数据采集脚本 (IK + 语言指令) | 本机 | 4-8h |
| 3.2 | 添加 domain randomization | 本机 | 2-4h |
| 3.3 | 采集 1000 episodes (自动化) | 本机 | 8-16h (GPU 运行) |
| 3.4 | 数据检查和清洗 | 本机 | 2h |
| 3.5 | Pi0 微调 (50K steps) | **Spark** | 8-16h |
| 3.6 | 仿真评估: 100 次随机位置 | 本机 | 1h |
| 3.7 | 如需要: 调参/增加数据/重新训练 | — | 视情况 |
| 3.8 | 仿真全流程: Pi0 策略完成一盘对弈 | 本机 | 2-4h |

**数据格式 (LeRobot dataset)：**
```
每帧: observation.images.top (320×240 RGB)
      observation.state (float32[6] 关节角度)
      action (float32[6] 关节目标)
      language_instruction ("把黑棋放到第3行第5列")
```

**关键检查点 (3.6 之后)：**
- [ ] Pi0 在仿真中成功率 > 85%
- [ ] 策略能理解不同表述的语言指令
- [ ] 策略能处理棋盘所有区域 (包括边角)

- 零样本成功率 < 30% → 需要回头加强 domain randomization

## 5. 风险应对矩阵

| 风险 | 概率 | 应对方案 | 触发条件 |
|------|------|---------|---------|
| 仿真物理抓取调不通 | 中 | 调整棋子摩擦/质量/夹爪力度; 参考 LeIsaac PickOrange 参数 | 2 周内成功率 < 80% |
| Pi0 训练效果不理想 | 中 | 退回 ACT (更简单); 或增加数据量; 或用 SmolVLA | 仿真评估成功率 < 70% |
| Sim-to-Real 差距大 | 中 | 加强 domain randomization; 真实数据微调 20-50 条 | 零样本成功率 < 30% |
| Spark 延迟到货 | 中 | 先用 ACT 在 RTX 4060 Ti 上训练验证流程 | — |
| 扁平棋子抓取困难 | 高 | 棋子加厚/加唇; 夹爪贴硅胶垫; 仿真中先验证 | 真实抓取成功率 < 80% |
| 语言指令理解偏差 | 低 | 限制模板种类; 或回退到坐标输入 (environment_state) | 语言泛化测试 < 90% |
