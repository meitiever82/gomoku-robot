# 五子棋机器人 — 实施计划

> 版本: V0.2 | 日期: 2026-03-19

## 1. 总体路线（仿真优先）

```
Phase 0        Phase 1        Phase 2          Phase 3          Phase 4          Phase 5
准备阶段  ──▶  软件验证  ──▶  仿真环境搭建  ──▶  仿真训练验证  ──▶  硬件迁移  ──▶  实物联调

文档/物料/     引擎/视觉      Isaac Sim        脚本化数据采集    标定/微调       完整对弈
环境搭建      单元测试       场景+任务搭建     ACT训练/评估     sim-to-real     调优迭代
(本机)        (本机)        (GPU 机器)       (GPU 机器)       (本机+硬件)     (本机+硬件)
```

**与原计划 (V0.1) 的核心变化：** 在硬件集成前插入仿真阶段，在 Isaac Sim 中完成数据采集和策略训练。

### Phase 2: 仿真环境搭建 (GPU 机器)

**目标：** 在 Isaac Sim 中搭建五子棋场景，跑通基础 pick-and-place。

**前置条件：** GPU 机器上安装好 Isaac Sim + LeIsaac (详见 `docs/04-仿真环境搭建指南.md`)

| 步骤 | 任务 | 预计工作量 |
|------|------|-----------|
| 2.1 | GPU 机器环境搭建 (conda + IsaacSim + LeIsaac) | 2-4h |
| 2.2 | 验证 LeIsaac SO-101 PickOrange 环境正常运行 | 1h |
| 2.3 | URDF → USD 转换 (SO-100 模型) | 1h |
| 2.4 | 创建棋盘/棋子/托盘 USD 资产 | 4-8h |
| 2.5 | 编写五子棋 IsaacLab 任务环境 (gomoku_env_cfg.py) | 8-16h |
| 2.6 | 仿真中 IK 抓放测试 | 2-4h |
| 2.7 | 仿真中视觉模块测试 (仿真相机 → 棋盘检测) | 2-4h |

**关键检查点 (2.6 之后)：**
- [ ] 仿真臂能从托盘拾取棋子
- [ ] 棋子能放到 9×9 棋盘任意位置
- [ ] 仿真相机能正确识别棋盘状态
- [ ] 单次 pick-and-place 仿真成功率 > 90%

**新增文件：**
```

```

### Phase 3: 仿真数据采集与训练 (GPU 机器)

**目标：** 脚本化生成大量训练数据，训练 ACT 策略，在仿真中验证。

| 步骤 | 任务 | 预计工作量 |
|------|------|-----------|
| 3.1 | 编写脚本化数据采集 (IK 轨迹生成) | 4-8h |
| 3.2 | 采集 500-1000 episodes (自动化) | 2-4h (GPU 运行) |
| 3.3 | 数据检查和清洗 | 2h |
| 3.4 | ACT 训练 (100k steps) | 4-8h (GPU) |
| 3.5 | 仿真评估: 100 次随机位置 pick-and-place | 1h |
| 3.6 | 如需要: 调参/增加数据/重新训练 | 视情况 |
| 3.7 | 仿真全流程联调: 完整对弈一盘 | 2-4h |

**脚本化数据采集优势：**
- 无需 Leader 臂，IK 自动生成轨迹
- 可轻松采集 1000+ 条数据 (vs 真实 30-50 条)
- 可加 domain randomization (光照、棋子位置微扰、摩擦系数)

**关键检查点 (3.5 之后)：**
- [ ] ACT 策略在仿真中成功率 > 85%
- [ ] 策略能处理棋盘所有区域 (包括边角)
- [ ] 完整对弈流程在仿真中跑通

### Phase 4: 硬件迁移 (本机 + 硬件)

**目标：** 将仿真中验证过的策略迁移到真实 SO-100。

**前置条件：** Phase 3 仿真验证通过 + 物料到位。

| 步骤 | 任务 | 预计工作量 |
|------|------|-----------|
| 4.1 | SO-100 连接和校准 (`lerobot-calibrate`) | 1h |
| 4.2 | 相机标定 (`gomoku-calibrate-board`) | 1h |
| 4.3 | 机器人标定 (`gomoku-calibrate-robot`) | 1h |
| 4.4 | 零样本测试: 直接用仿真策略在真实臂上测试 | 2h |
| 4.5 | 评估 sim-to-real 差距 | — |
| 4.6 | (如需要) 采集 10-20 条真实示教数据，微调策略 | 4-8h |
| 4.7 | 真实环境视觉检测调参 (HSV 阈值) | 1-2h |

**Sim-to-Real 成功标准：**
- 零样本迁移成功率 > 50% → 可接受，微调即可
- 零样本迁移成功率 < 30% → 需要分析差距原因，可能需要更多 domain randomization

### Phase 5: 实物联调

**目标：** 完整对弈流程在真实硬件上跑通。

与原 V0.1 的 Phase 4 相同，但此时策略已经在仿真中充分验证，主要工作是：

| 步骤 | 任务 |
|------|------|
| 5.1 | 集成: 主循环连接所有模块 |
| 5.2 | 冒烟测试: AI 下 3 步，人下 3 步 |
| 5.3 | 全局对弈: 完成一盘棋 |
| 5.4 | 问题记录和修复 |
| 5.5 | 参数调优: 夹爪力度、HSV 阈值、控制频率 |
| 5.6 | 稳定性测试: 连续 3 盘棋 |

## 5. 风险应对矩阵 (更新)

| 风险 | 概率 | 应对方案 | 触发条件 |
|------|------|---------|---------|
| Isaac Sim 安装/兼容问题 | 中 | 退回 MuJoCo 方案或用 NVIDIA Brev 云端 | 安装超过 1 天未解决 |
| 仿真场景搭建耗时过长 | 中 | 先基于 LeIsaac PickOrange 改造而非从零开始 | 超过 2 周无进展 |
| Sim-to-Real 迁移差距大 | 中 | 增加 domain randomization; 真实数据微调 | 零样本成功率 < 30% |
| ACT 策略泛化不足 | 中 | 仿真中大量数据弥补; 退回 IK fallback | 成功率 < 70% after 1000 demos |
| 扁平棋子抓取困难 | 高 | 重新设计棋子（加厚/加唇）; 仿真中先验证 | 仿真中成功率 < 80% |
| SO-100 到货延迟 | 中 | 仿真阶段不受影响，继续推进 | — |
| GPU 机器不可用 | 低 | NVIDIA Brev 云端; 或本机 MuJoCo | — |
